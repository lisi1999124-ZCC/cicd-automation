# CI/CD 流水线架构

## 概述
本文档描述面向生产部署的自动化 CI/CD 流水线架构。

## 流水线阶段

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           CI/CD 流水线流程                                       │
└─────────────────────────────────────────────────────────────────────────────────┘

  [开发者] 
      │
      ▼
┌─────────────────┐
│  代码提交       │  ← Git Push / Pull Request
│  (源码阶段)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  安全扫描       │  ← Trivy / SAST / 依赖检查
│  (构建前)       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  构建与测试     │  ← Docker 构建 / 单元测试 / 集成测试
│  (构建阶段)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  镜像推送        │  ← Docker 仓库 / 制品存储
│  (推送阶段)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  部署预发布     │  ← 蓝绿部署 / 金丝雀部署
│  (预发布环境)   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  冒烟测试       │  ← 健康检查 / 集成测试
│  (验证阶段)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  部署生产       │  ← 滚动更新 / 蓝绿切换
│  (生产环境)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  监控与告警     │  ← Prometheus / Grafana / PagerDuty
│  (可观测性)    │
└─────────────────┘
```

## 环境策略

| 环境 | 用途 | 部署方式 | 自动扩缩容 |
|------|------|----------|------------|
| 开发环境 | 功能测试 | 按需部署 | 否 |
| 预发布环境 | 上线前验证 | 自动部署 | 否 |
| 生产环境 | 线上流量 | 手动审批 | 是 |

## 部署策略

### 蓝绿部署
```
┌─────────────────┐      ┌─────────────────┐
│  绿色 (线上)    │◄────►│  蓝色 (待命)    │
│  100% 流量      │      │  0% 流量         │
└────────┬────────┘      └────────┬────────┘
         │                        │
         └────────┬───────────────┘
                  ▼
         ┌─────────────────┐
         │   负载均衡器    │
         └────────┬────────┘
                  │
         ┌────────┴────────┐
         │                 │
    验证后切换到蓝色   切换前健康检查
```

### 金丝雀部署
```
┌────────────────────────────────────────┐
│         负载均衡器                     │
│  ┌──────────────────────────────────┐  │
│  │  5% → 金丝雀    │  95% → 稳定版 │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
         │                    │
         ▼                    ▼
┌─────────────────┐   ┌─────────────────┐
│  金丝雀 (5%)    │   │  稳定版 (95%)   │
│  新版本         │   │  当前版本       │
└─────────────────┘   └─────────────────┘
```

## 质量关卡

| 阶段 | 检查项 | 标准 |
|------|--------|------|
| 安全扫描 | Trivy | 0 个严重漏洞 |
| 单元测试 | Jest/Mocha | 覆盖率 ≥ 80% |
| 集成测试 | Postman/Newman | 全部通过 |
| 预发布部署 | 健康检查 | HTTP 200 |
| 生产部署 | 手动审批 | 必须审批 |

## 回滚策略

1. **自动回滚**：健康检查失败时自动触发
2. **手动回滚**：通过 GitHub Actions 手动调度
3. **一键回滚**：`kubectl rollout undo deployment/app`

## 通知渠道

- ✅ 成功：Slack #deployments
- ⚠️ 警告：Slack #deployments-alerts
- 🚨 失败：PagerDuty（值班通知）

## 跟踪指标

- 部署频率（每日）
- 变更前置时间（提交到上线）
- 变更失败率（%）
- 平均恢复时间（MTTR）
